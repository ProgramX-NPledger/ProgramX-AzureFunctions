
## CORS

The headers for CORS implementation are not natively generated by Azure Functions. Instead the following configuration is required for local/production configureation.

### Development

In `local.settings.json`:

```json
{
  "Host": {
    "CORS": "http://localhost:4200,https://localhost:4200",
    "CORSCredentials": false
  }
}
```

The Azure Functions host command can be forced to generate the required `Access-Control-Allow-Origin` header by modifying its parameters:

`host start --pause-on-error --port 7276 --cors http://localhost:4200`

The `--cors` parameter is a comma-separated list of allowed origins.

To allow CORS credentials, add the `--cors-credentials` parameter.

### Production

In Azure:

In the Function App in Azure:
Function App → API → CORS
Add http://localhost:4200 (for development) and https://whatever.com (production).
